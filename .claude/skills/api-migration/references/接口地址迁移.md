# 接口地址迁移规范

## 1. 旧项目接口地址来源

**重要**：迁移 API 接口时，必须参考旧项目的 URL 常量文件获取正确的接口地址。

### 1.1 URL 常量文件位置

```plain
gitee-example/constant/url.js
```

### 1.2 文件结构说明


## 2. 迁移步骤

### 2.1 第一步：查找旧项目接口地址

在迁移某个模块的 API 时，**必须先**在 `gitee-example/constant/url.js` 中查找对应的接口地址：

```bash
# 示例：查找维修相关接口
Grep: pattern="Repair|repair" path="gitee-example/constant/url.js"
```

### 2.2 第二步：提取接口路径

从 URL 常量中提取实际的接口路径（去除 `baseUrl +` 部分）：

| 旧项目常量定义                                          | 提取的接口路径                               |
| :------------------------------------------------------ | :------------------------------------------- |
| `baseUrl + "app/ownerRepair.listOwnerRepairs"`          | `/app/ownerRepair.listOwnerRepairs`          |
| `baseUrl + "app/inspectionTask.listInspectionTasks"`    | `/app/inspectionTask.listInspectionTasks`    |
| `baseUrl + "app/maintainanceTask.listMaintainanceTask"` | `/app/maintainanceTask.listMaintainanceTask` |

### 2.3 第三步：在新项目中定义 API

```typescript
// src/api/repair.ts
import { http } from "@/http/alova";
import type { PaginationResponse } from "@/types/api";
import type { RepairOrder } from "@/types/repair";

/** 查询维修工单列表 */
export function getRepairOrderList(params: RepairListParams) {
	// ✅ 使用从 url.js 提取的接口路径
	return http.Get<PaginationResponse<RepairOrder>>("/app/ownerRepair.listOwnerRepairs", { params });
}
```

```typescript
			// Mock 实现...
		},
	},
]);
```

## 4. 注意事项

### 4.1 URL 前缀处理

- **旧项目**：使用 `baseUrl + "app/..."` 拼接
- **新项目 API 定义**：直接使用 `/app/...` 路径
- **新项目 Mock 定义**：使用 `defineUniAppMock`，它会自动处理前缀

### 4.2 特殊路径处理

部分接口使用 `callComponent` 前缀而非 `app` 前缀：

```javascript
// 旧项目
uploadImage: baseUrl + "callComponent/uploadFile/uploadPhotoImage";
queryDictInfo: baseUrl + "callComponent/core/list";
```

迁移时保持原有路径结构：

```typescript
// 新项目
export function uploadImage(data: FormData) {
	return http.Post("/callComponent/uploadFile/uploadPhotoImage", data);
}
```

### 4.3 接口命名规范

| 旧项目命名风格               | 新项目命名风格           |
| :--------------------------- | :----------------------- |
| `listOwnerRepairs`           | `getRepairOrderList`     |
| `saveOwnerRepair`            | `createRepairOrder`      |
| `updateInspectionTaskDetail` | `updateInspectionDetail` |

建议：

- 查询列表：`get{Entity}List`
- 查询详情：`get{Entity}Detail`
- 创建：`create{Entity}`
- 更新：`update{Entity}`
- 删除：`delete{Entity}`
