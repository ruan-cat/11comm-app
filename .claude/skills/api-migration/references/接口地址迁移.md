# 接口地址迁移规范

## 1. 旧项目接口地址来源

迁移接口时，必须先参考：

```plain
gitee-example/constant/url.js
```

这里仍然是当前仓库最可靠的旧业务路径来源。

## 2. 当前迁移步骤

### 2.1. 第一步：在旧项目查路径

例如旧项目里：

```typescript
listOwnerRepairs: baseUrl + "app/ownerRepair.listOwnerRepairs";
```

提取后得到：

```typescript
"/app/ownerRepair.listOwnerRepairs";
```

### 2.2. 第二步：写前端 API

```typescript
export function getRepairOrderList(params: RepairListParams) {
	return http.Get<RepairListResponse>("/app/ownerRepair.listOwnerRepairs", { params });
}
```

### 2.3. 第三步：写共享 endpoint

```typescript
{
  url: '/app/ownerRepair.listOwnerRepairs',
  method: ['GET', 'POST'],
  handler: ({ params }) => {
    const result = repository.list(...)
    return successResponse({
      ownerRepairs: result.list,
      total: result.total,
      page: result.page,
      row: result.pageSize,
    }, '查询成功')
  },
}
```

### 2.4. 第四步：补 H5 mock 薄包装

```typescript
export default defineUniAppMock(createLegacyMockDefinitionsFromEndpoints(repairEndpointDefinitions));
```

注意：当前仓库的 H5 mock 层不再要求你重新手写一遍接口 URL 与业务逻辑。

## 3. 新旧模式区别

|       环节       |                旧误导流程                |                           当前推荐流程                            |
| :--------------: | :--------------------------------------: | :---------------------------------------------------------------: |
|     URL 来源     |            看旧项目 `url.js`             |                       仍然看旧项目 `url.js`                       |
|     API 定义     |          `src/api/{module}.ts`           |                       `src/api/{module}.ts`                       |
|  mock 业务实现   |     `src/api/mock/{module}.mock.ts`      |     `server/modules/{module}/endpoints.ts` + `repository.ts`      |
| H5 mock 导出方式 | `defineUniAppMock([...业务 handler...])` | `defineUniAppMock(createLegacyMockDefinitionsFromEndpoints(...))` |
|  Nitro 接入方式  |         再补一套单独的 `/api/**`         |             通过 `legacy-dispatch` 继续承接旧业务路径             |

## 5. 注意事项

### 5.1. 不要统一改成 /api/

当前仓库为了兼容旧前端契约，路径仍然保持 `/app/**` 与 `/callComponent/**`。

### 5.2. 不要在 URL 里补 `/dev-api`

H5 mock 的前缀由运行时注入，不属于业务路径本身。

### 5.3. 共享化模块的 URL 只维护一处

共享化后，URL 的权威定义在 `server/modules/*/endpoints.ts`，而 `src/api/mock/*.mock.ts` 只是从共享 endpoint 派生出来的 Vite mock 定义。
