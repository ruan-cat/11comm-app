# 静态资源导入规范

## 1. 问题背景

在 uni-app + Vite 项目中，静态资源（如图片）的导入方式与普通 Vue 项目有所不同。错误的导入方式会导致图片无法显示。

## 2. 常见错误

### 2.1 ❌ 错误：在模板中直接使用路径别名字符串

```vue
<script setup>
/** 顶部入口列表 - 错误写法 */
const headerEntries = [
	{ id: "complaint", name: "投诉待办", icon: "@/static/image/index/i_complaint.png" },
	{ id: "repair", name: "报修待办", icon: "@/static/image/index/i_repair.png" },
];
</script>

<template>
	<!-- ❌ 错误：@/ 别名在模板动态绑定中不会被解析 -->
	<image v-for="entry in headerEntries" :key="entry.id" :src="entry.icon" />
</template>
```

**问题原因**：Vite 只会在 JavaScript/TypeScript 的 `import` 语句中解析 `@/` 路径别名。在模板的 `:src` 动态绑定中，`@/static/...` 只是一个普通字符串，不会被转换为实际路径。

## 3. 正确的导入方式

### 3.1 ✅ 方式一：使用 import 导入（推荐）

```vue
<script setup>
/** 导入静态图片资源 */
import iComplaint from "@/static/image/index/i_complaint.png";
import iRepair from "@/static/image/index/i_repair.png";
import iInspection from "@/static/image/index/i_inspection.png";
import iMachine from "@/static/image/index/i_machine.png";

/** 顶部入口列表 - 正确写法 */
const headerEntries = [
	{ id: "complaint", name: "投诉待办", icon: iComplaint },
	{ id: "repair", name: "报修待办", icon: iRepair },
	{ id: "inspection", name: "巡检打卡", icon: iInspection },
	{ id: "maintenance", name: "设备保养", icon: iMachine },
];
</script>

<template>
	<!-- ✅ 正确：使用导入的变量 -->
	<image v-for="entry in headerEntries" :key="entry.id" :src="entry.icon" />
</template>
```

### 3.2 ✅ 方式二：使用相对路径（静态字符串）

```vue
<template>
	<!-- ✅ 正确：静态 src 属性使用相对路径 -->
	<image src="/static/image/index/i_complaint.png" />

	<!-- 或者使用相对于当前文件的路径 -->
	<image src="../../static/image/index/i_complaint.png" />
</template>
```

### 3.3 ✅ 方式三：使用 /static 绝对路径

在 uni-app 中，`/static` 目录下的资源可以直接通过绝对路径访问：

```vue
<script setup>
/** 使用 /static 绝对路径 */
const headerEntries = [
	{ id: "complaint", name: "投诉待办", icon: "/static/image/index/i_complaint.png" },
	{ id: "repair", name: "报修待办", icon: "/static/image/index/i_repair.png" },
];
</script>

<template>
	<image v-for="entry in headerEntries" :key="entry.id" :src="entry.icon" />
</template>
```

## 4. 使用场景选择

| 场景             | 推荐方式                   | 说明                 |
| :--------------- | :------------------------- | :------------------- |
| 单个静态图片     | 方式二（相对路径）         | 简单直接             |
| 多个图片配置数组 | 方式一（import 导入）      | 类型安全，IDE 支持好 |
| 动态图片路径     | 方式三（/static 绝对路径） | 灵活，但无编译时检查 |
| 远程图片 URL     | 直接使用 URL 字符串        | 无需特殊处理         |

## 5. 注意事项

### 5.1 TypeScript 类型声明

如果使用 TypeScript，需要确保有图片模块的类型声明。项目中应该已有 `src/types/async-import.d.ts`：

```typescript
/** 图片资源模块声明 */
declare module "*.png" {
	const src: string;
	export default src;
}

declare module "*.jpg" {
	const src: string;
	export default src;
}

declare module "*.jpeg" {
	const src: string;
	export default src;
}

declare module "*.gif" {
	const src: string;
	export default src;
}

declare module "*.svg" {
	const src: string;
	export default src;
}
```

### 5.2 与 getImageUrl 的区别

- **本地静态资源**：使用本文档描述的导入方式
- **远程/动态图片**：使用 `getImageUrl` 函数处理（详见 `工具函数迁移.md`）

```typescript
import { getImageUrl } from "@/utils";

/** 本地静态图片 - 使用 import */
import localIcon from "@/static/image/icon.png";

/** 远程/动态图片 - 使用 getImageUrl */
const remoteImage = getImageUrl(activity.headerImg, communityId);
```
