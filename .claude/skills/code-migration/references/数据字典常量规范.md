# 数据字典常量使用规范

## 1. 核心规范

### 1.1 基本原则

1. **集中管理**: 所有枚举/下拉字典存放于 `src/constants/{模块}.ts`
2. **统一类型**: 类型统一使用 `ColumnItem[]`（来自 `wot-design-uni/components/wd-picker-view/types`）
3. **字符串值**: `value` 必须是字符串类型（禁止混用 number/boolean）
4. **展示文案**: `label` 用于展示名称
5. **相对路径**: mock 文件引用常量使用相对路径（避免别名解析失败）

### 1.2 命名规范

- **文件命名**: 小写 + 短横线，如 `repair.ts`、`complaint.ts`、`property-application.ts`
- **常量命名**: 全大写 + 下划线，如 `REPAIR_STATUSES`、`COMPLAINT_TYPE_OPTIONS`
- **通用常量**: 放在 `src/constants/common.ts`
- **业务常量**: 放在各自模块文件

### 3.2 在 Mock 文件中使用

**重要**: Mock 文件中引用常量必须使用相对路径！

```typescript
// src/api/mock/repair.mock.ts
import { defineUniAppMock, successResponse } from "./shared/utils";
// ✅ 正确：使用相对路径
import { REPAIR_STATUSES, REPAIR_TYPES } from "../../constants/repair";

// ❌ 错误：使用路径别名（Vite Mock 插件无法解析）
// import { REPAIR_STATUSES, REPAIR_TYPES } from '@/constants/repair'

const mockRepairDatabase = {
	repairs: [] as RepairOrder[],

	/** 创建模拟维修工单 */
	createMockRepair(id: string): RepairOrder {
		// 从状态常量中随机选择
		const statusItem = REPAIR_STATUSES[Math.floor(Math.random() * REPAIR_STATUSES.length)];

		// 从类型常量中随机选择
		const typeItem = REPAIR_TYPES[Math.floor(Math.random() * REPAIR_TYPES.length)];

		return {
			repairId: `REP_${id}`,
			title: `维修工单${id}`,
			// ✅ value 存储字段值（字符串）
			statusCd: statusItem.value,
			// ✅ label 用于显示名称
			statusName: statusItem.label,
			repairType: typeItem.value,
			repairTypeName: typeItem.label,
			createTime: new Date().toISOString(),
			updateTime: new Date().toISOString(),
		};
	},
};

export default defineUniAppMock([
	{
		url: "/app/ownerRepair.listOwnerRepairs",
		method: "POST",
		body: () => {
			const repairs = mockRepairDatabase.repairs.map((repair) => {
				// 从常量中查找对应的 label
				const statusItem = REPAIR_STATUSES.find((s) => s.value === repair.statusCd);
				const typeItem = REPAIR_TYPES.find((t) => t.value === repair.repairType);

				return {
					...repair,
					statusName: statusItem?.label || "",
					repairTypeName: typeItem?.label || "",
				};
			});

			return successResponse({
				ownerRepairs: repairs,
				total: repairs.length,
			});
		},
	},
]);
```

### 3.3 TypeScript 类型与常量结合

```typescript
// types/repair.ts
import type { REPAIR_STATUSES, REPAIR_TYPES } from "@/constants/repair";

/**
 * 维修工单状态类型（从常量推导）
 */
export type RepairStatus = (typeof REPAIR_STATUSES)[number]["value"];
// 等价于: '10001' | '10002' | '10003' | '10004' | '10005'

/**
 * 维修类型（从常量推导）
 */
export type RepairType = (typeof REPAIR_TYPES)[number]["value"];
// 等价于: '1' | '2' | '3' | '4'

/**
 * 维修工单接口
 */
export interface RepairOrder {
	repairId: string;
	title: string;
	description: string;
	statusCd: RepairStatus; // 使用推导的联合类型
	statusName: string;
	repairType: RepairType; // 使用推导的联合类型
	repairTypeName: string;
	createTime: string;
	updateTime: string;
}
```

## 5. 工具函数

### 5.1 获取 Label 工具函数

```typescript
// utils/constants.ts
import type { ColumnItem } from "wot-design-uni/components/wd-picker-view/types";

/**
 * 根据 value 获取 label
 * @param options 选项数组
 * @param value 值
 * @param defaultLabel 默认标签（未找到时返回）
 * @example getLabelByValue(REPAIR_STATUSES, '10001') // '待派单'
 */
export function getLabelByValue(options: ColumnItem[], value: string, defaultLabel: string = "-"): string {
	const item = options.find((option) => option.value === value);
	return item?.label || defaultLabel;
}

/**
 * 根据 value 获取完整的 ColumnItem
 * @example getItemByValue(REPAIR_STATUSES, '10001')
 */
export function getItemByValue(options: ColumnItem[], value: string): ColumnItem | undefined {
	return options.find((option) => option.value === value);
}
```

### 5.2 使用示例

```typescript
<script setup lang="ts">
import { REPAIR_STATUSES } from '@/constants/repair'
import { getLabelByValue } from '@/utils/constants'

const repair = ref({
  repairId: 'REP_001',
  statusCd: '10001'
})

// 获取状态标签
const statusLabel = computed(() =>
  getLabelByValue(REPAIR_STATUSES, repair.value.statusCd)
)
// 结果: '待派单'
</script>

<template>
  <view>状态: {{ statusLabel }}</view>
</template>
```

## 6. 常见问题和最佳实践

### 6.1 ❌ 错误示例

```typescript
// ❌ 错误：value 使用 number 类型
export const WRONG_STATUSES = [
	{ value: 1, label: "待处理" }, // 应该是字符串 '1'
	{ value: 2, label: "已完成" }, // 应该是字符串 '2'
];

// ❌ 错误：value 使用 boolean 类型
export const WRONG_FLAGS = [
	{ value: true, label: "是" }, // 应该是字符串 '1'
	{ value: false, label: "否" }, // 应该是字符串 '0'
];

// ❌ 错误：混用不同类型
export const MIXED_TYPES = [
	{ value: "1", label: "选项1" },
	{ value: 2, label: "选项2" }, // 类型不一致
	{ value: true, label: "选项3" }, // 类型不一致
];

// ❌ 错误：Mock 文件中使用路径别名
import { REPAIR_STATUSES } from "@/constants/repair"; // Vite Mock 插件无法解析
```
