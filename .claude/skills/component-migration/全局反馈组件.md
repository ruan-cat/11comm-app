# 全局反馈组件使用指南

## 核心原则

本项目采用**全局组件模式**实现 Toast/Message/Loading 反馈功能,所有反馈组件统一在 `App.vue` 根组件注册,避免子组件内挂载导致的层级遮挡问题。

## ⚠️ 严格禁止

以下做法**严格禁止**,会导致层级遮挡、重复渲染等问题:

- ❌ 在子组件内直接使用 `<wd-toast />`
- ❌ 在子组件内直接使用 `<wd-message-box />`
- ❌ 在子组件内直接使用 `<wd-loading />`
- ❌ 在子组件内通过 `useToast('globalToast')` 等方式挂载局部反馈组件
- ❌ 手动管理 loading 状态

## ✅ 必须使用

必须使用以下全局组合式函数:

- `useGlobalToast()` - 全局 Toast 提示
- `useGlobalMessage()` - 全局 Message 对话框
- `useGlobalLoading()` - 全局 Loading 加载

## 1. Toast 提示组件

### 迁移对比

**❌ 旧代码 (子组件内使用 Toast)**:

```vue
<script setup lang="ts">
import { useToast } from "wot-design-uni";

const toast = useToast("globalToast");

function showSuccess() {
	toast.success("操作成功");
}
</script>

<template>
	<!-- ❌ 错误: 在子组件内挂载 Toast -->
	<wd-toast id="globalToast" />
	<wd-button @click="showSuccess">显示提示</wd-button>
</template>
```

**✅ 新代码 (全局组合式 API)**:

```vue
<script setup lang="ts">
import { useGlobalToast } from "@/hooks/useGlobalToast";

const toast = useGlobalToast();

function showSuccess() {
	toast.success("操作成功");
}
</script>

<template>
	<!-- ✅ 正确: 无需在子组件内挂载 Toast 组件 -->
	<wd-button @click="showSuccess">显示提示</wd-button>
</template>
```

### Toast 使用方法

```typescript
import { useGlobalToast } from "@/hooks/useGlobalToast";

const toast = useGlobalToast();

// 成功提示
toast.success("操作成功");

// 错误提示
toast.error("操作失败");

// 警告提示
toast.warning("请注意");

// 信息提示
toast.info("提示信息");

// 自定义配置
toast.show({
	msg: "自定义消息",
	duration: 3000,
	iconName: "success",
	position: "top",
});
```

### Toast 特性

- ✅ 支持 success/error/info/warning 快捷方法
- ✅ 自动页面路径检测
- ✅ 自定义图标、位置、持续时间
- ✅ 全局单例,避免重复显示

## 2. Message 对话框组件

### 迁移对比

**❌ 旧代码**:

```vue
<script setup lang="ts">
import { useMessage } from "wot-design-uni";

const message = useMessage("globalMessage");

function showConfirm() {
	message.confirm({
		msg: "确认删除?",
		success: () => {
			console.log("确认");
		},
	});
}
</script>

<template>
	<!-- ❌ 错误: 在子组件内挂载 Message -->
	<wd-message-box id="globalMessage" />
	<wd-button @click="showConfirm">确认对话框</wd-button>
</template>
```

**✅ 新代码**:

```vue
<script setup lang="ts">
import { useGlobalMessage } from "@/hooks/useGlobalMessage";

const message = useGlobalMessage();

function showConfirm() {
	message.confirm({
		msg: "确认删除?",
		success: () => {
			console.log("确认");
		},
	});
}
</script>

<template>
	<!-- ✅ 正确: 无需在子组件内挂载 Message 组件 -->
	<wd-button @click="showConfirm">确认对话框</wd-button>
</template>
```

### Message 使用方法

```typescript
import { useGlobalMessage } from "@/hooks/useGlobalMessage";

const message = useGlobalMessage();

// 警告对话框
message.alert({
	msg: "这是一个警告",
	success: () => {
		console.log("已确认");
	},
});

// 确认对话框
message.confirm({
	msg: "确认删除?",
	success: () => {
		console.log("已确认");
	},
	fail: () => {
		console.log("已取消");
	},
});

// 输入对话框
message.prompt({
	msg: "请输入内容",
	success: (value) => {
		console.log("输入值:", value);
	},
});

// 自定义配置
message.confirm({
	title: "删除确认",
	msg: "确认删除这条数据?",
	confirmButtonText: "删除",
	cancelButtonText: "取消",
	success: () => {
		// 确认回调
	},
	fail: () => {
		// 取消回调
	},
});
```

### Message 特性

- ✅ 支持 alert/confirm/prompt 对话框
- ✅ success/fail 回调处理
- ✅ 自定义按钮样式
- ✅ 全局单例,避免重复显示

## 3. Loading 加载组件

### 迁移对比

**❌ 旧代码 (手动管理 loading 状态)**:

```vue
<script setup lang="ts">
import { ref } from "vue";

const loading = ref(false);

async function loadData() {
	loading.value = true;
	try {
		await fetchData();
	} finally {
		loading.value = false;
	}
}
</script>

<template>
	<!-- ❌ 错误: 手动管理loading状态 -->
	<wd-loading v-if="loading" />
	<view v-else>
		<!-- 内容 -->
	</view>
</template>
```

**✅ 新代码 (全局 Loading)**:

```vue
<script setup lang="ts">
import { useGlobalLoading } from "@/hooks/useGlobalLoading";

const loading = useGlobalLoading();

async function loadData() {
	loading.show("加载中...");
	try {
		await fetchData();
	} finally {
		loading.hide();
	}
}
</script>

<template>
	<!-- ✅ 正确: 无需手动管理loading状态 -->
	<view>
		<!-- 内容 -->
	</view>
</template>
```

### Loading 使用方法

```typescript
import { useGlobalLoading } from "@/hooks/useGlobalLoading";

const loading = useGlobalLoading();

// 显示加载
loading.show("加载中...");

// 隐藏加载
loading.hide();

// 自动隐藏（3秒后）
loading.show("加载中...", 3000);

// 异步操作封装
async function loadData() {
	loading.show("正在加载数据...");
	try {
		const result = await fetchData();
		// 处理结果
	} catch (error) {
		console.error("加载失败:", error);
	} finally {
		loading.hide();
	}
}
```

### Loading 特性

- ✅ 自动显示遮罩层
- ✅ 支持自定义加载文案
- ✅ 统一管理加载状态
- ✅ 自动防止重复显示

## 完整使用示例

### 示例 1: 表单提交

```vue
<script setup lang="ts">
import { ref } from "vue";
import { useGlobalToast } from "@/hooks/useGlobalToast";
import { useGlobalLoading } from "@/hooks/useGlobalLoading";
import { submitForm } from "@/api/form";

const toast = useGlobalToast();
const loading = useGlobalLoading();

const formData = ref({
	name: "",
	phone: "",
});

async function handleSubmit() {
	// 验证
	if (!formData.value.name) {
		toast.warning("请输入姓名");
		return;
	}

	// 提交
	loading.show("提交中...");
	try {
		await submitForm(formData.value);
		toast.success("提交成功");
		// 返回上一页或其他操作
	} catch (error) {
		toast.error("提交失败");
	} finally {
		loading.hide();
	}
}
</script>
```

### 示例 2: 删除确认

```vue
<script setup lang="ts">
import { useGlobalMessage } from "@/hooks/useGlobalMessage";
import { useGlobalToast } from "@/hooks/useGlobalToast";
import { useGlobalLoading } from "@/hooks/useGlobalLoading";
import { deleteItem } from "@/api/item";

const message = useGlobalMessage();
const toast = useGlobalToast();
const loading = useGlobalLoading();

function handleDelete(itemId: string) {
	message.confirm({
		title: "删除确认",
		msg: "确认删除这条数据?",
		confirmButtonText: "删除",
		success: async () => {
			loading.show("删除中...");
			try {
				await deleteItem(itemId);
				toast.success("删除成功");
				// 刷新列表
				refreshList();
			} catch (error) {
				toast.error("删除失败");
			} finally {
				loading.hide();
			}
		},
	});
}
</script>
```

### 示例 3: 数据加载

```vue
<script setup lang="ts">
import { ref, onMounted } from "vue";
import { useGlobalToast } from "@/hooks/useGlobalToast";
import { useGlobalLoading } from "@/hooks/useGlobalLoading";
import { getDataList } from "@/api/data";

const toast = useGlobalToast();
const loading = useGlobalLoading();

const dataList = ref([]);

async function loadData() {
	loading.show("加载数据中...");
	try {
		const result = await getDataList();
		dataList.value = result.list;

		if (result.list.length === 0) {
			toast.info("暂无数据");
		}
	} catch (error) {
		toast.error("加载失败,请重试");
	} finally {
		loading.hide();
	}
}

onMounted(() => {
	loadData();
});
</script>
```

## 常见问题

### 1. 为什么不能在子组件内使用反馈组件?

**答**: 在子组件内使用会导致:

- 层级遮挡问题(被其他组件遮挡)
- 重复渲染(多个页面同时挂载)
- 状态管理混乱(多个实例互相干扰)

全局组件模式解决了这些问题,统一在根组件管理。

### 2. 如何在多个页面使用同一个 Toast?

**答**: 使用 `useGlobalToast()` 获取的是同一个全局实例,在任何页面调用都会显示同一个 Toast。

### 3. Loading 显示后忘记隐藏怎么办?

**答**: 始终在 `finally` 块中调用 `loading.hide()`,确保无论成功还是失败都会隐藏 Loading:

```typescript
try {
	loading.show();
	await doSomething();
} finally {
	loading.hide(); // 必须在 finally 中
}
```

### 4. 如何自定义 Toast 样式?

**答**: 查看 `src/components/global/toast/README.md` 了解完整配置选项,包括位置、图标、颜色等。

通过统一的全局反馈组件,确保项目用户体验的一致性和代码的可维护性！
