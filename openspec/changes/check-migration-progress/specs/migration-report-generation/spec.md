## ADDED Requirements

### Requirement: 报告必须写入指定文件

检查结果 SHALL 写入 `docs/reports/2026-03-03-migration-progress-report.md`，在现有内容基础上进行扩充和修订。

#### Scenario: 报告文件写入成功

- **WHEN** 所有扫描和检查子代理完成工作
- **THEN** 报告文件被成功写入或更新，文件内容包含本次检查的完整结论

### Requirement: 报告必须包含迁移存在性统计

报告 SHALL 包含按模块分组的迁移存在性统计数据。

#### Scenario: 展示总体统计数据

- **WHEN** 报告被生成
- **THEN** 报告顶部须有总体统计表格，包含「已迁移 / 未迁移 / 不完整」三列数据及占比

#### Scenario: 展示每模块详细状态

- **WHEN** 报告被生成
- **THEN** 每个模块下须列出所有目标文件，并标注 ✅ / ❌ / ⚠️ 状态

### Requirement: 报告必须包含 Skills 合规性检查结果

报告 SHALL 为每个已迁移文件提供 Skills 合规性检查摘要。

#### Scenario: 逐文件展示合规性评级

- **WHEN** 报告被生成
- **THEN** 每个已迁移文件须有一行检查结果，包含各检查维度的 Pass / Warn / Fail / N/A 评级

#### Scenario: 汇总不合规问题

- **WHEN** 报告被生成
- **THEN** 报告中须有一个「不合规问题汇总」章节，列出所有 Fail 级别的问题及对应文件

### Requirement: 报告遵循项目报告编写规范

报告 SHALL 遵循 `AGENTS.md` 第 6 节「报告编写规范」中定义的格式要求。

#### Scenario: 报告一级标题包含日期

- **WHEN** 报告被生成
- **THEN** 一级标题格式为 `# 2026-03-03 <报告名称>`

#### Scenario: 报告表格居中对齐

- **WHEN** 报告中包含 Markdown 表格
- **THEN** 所有表格的对齐方式均为居中对齐（`:---:`）

#### Scenario: 报告多级标题包含数字序号

- **WHEN** 报告包含二级及以下标题
- **THEN** 所有二级、三级、四级标题均带有数字序号（如 `## 1. xxx`、`### 1.1 xxx`）

### Requirement: 报告必须提供未迁移模块的优先级建议

报告 SHALL 在结尾提供未迁移模块的迁移优先级排序建议。

#### Scenario: 展示优先级建议

- **WHEN** 报告被生成
- **THEN** 报告末尾须有「后续建议」章节，按高/中/低优先级列出未迁移模块及其建议迁移顺序
